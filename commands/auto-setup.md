**Step 1: 프로젝트 분석**

Agent 도구를 사용하여 setup-analyzer 에이전트를 호출하세요. 프롬프트: "현재 프로젝트를 분석해주세요. Glob으로 루트의 설정 파일들을 찾고, Read로 내용을 확인하세요. 주요 디렉토리 구조도 파악하세요. 기존 CLAUDE.md와 .claude/ 디렉토리도 확인하세요. 구조화된 분석 결과를 반환해주세요."

분석 결과를 받으면 사용자에게 보여주세요. 기존 .claude/ 설정이 있으면 사용자에게 처리 방식을 물어보세요 (덮어쓰기/머지/스킵).

**Step 2: 맞춤 인터뷰**

분석 결과를 기반으로, AskUserQuestion 도구를 사용하여 사용자에게 직접 질문하세요. 분석된 프레임워크에 맞는 질문 3-4개를 하세요. 모든 질문은 한국어로, 구체적 선택지를 포함하세요.

필수 질문: "이 프로젝트에서 Claude에게 주로 시킬 작업은?" / "꼭 지켜야 하는 코딩 규칙이 있나요?"
프레임워크별 추가 질문: Spring Boot면 JPA/배포 관련, Next.js면 상태관리/SSR 관련, Express면 인증/DB 관련 질문을 하세요.
핵심 질문 후 "더 세부 설정을 하시겠어요?"를 물어보세요.

**Step 3: 파일 생성**

Agent 도구를 사용하여 setup-generator 에이전트를 호출하세요. 프롬프트에 Step 1 분석 결과 전체와 Step 2 인터뷰 답변 전체를 포함하세요. 생성 대상: CLAUDE.md, .claude/rules/, .claude/skills/ (인터뷰에서 파악된 것만).

**Step 4: 검증**

Agent 도구를 사용하여 setup-reviewer 에이전트를 호출하세요. 생성된 파일을 검증하고, 문제가 있으면 setup-generator를 다시 호출하여 수정하세요 (최대 2회).

**Step 5: 완료 보고**
